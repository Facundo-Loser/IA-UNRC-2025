Relación entre hiper-parámetros y overfitting/undefitting en distintos modelos:

- Regresión:*Undefitting: se produce suponer que los datos tenian una forma funcional y en realidad tenian otra (sesgo). Tambien por utilizar 1 o pocos grados de libertad cuando los datos no tienen una forma lineal (según yo).
            *Overfitting: se produce por dejar muchos grados de libertad en reg polinomial (varianza). También por el error irreducible y por entrenar el modelo muchas épocas y no realizar early stopping (se produce overfitting en las últimas iteraciones). Si no se realiza regularización (ridge,lasso o elastic net) también puede afectar y caer en overfitting.

-Clasificación:*Underfitting:
                -en k-nearest-Neighbors: usar un k=1 o muy chico.
                -SVM: usar un C muy bajo softmargin (permitir muchos errores). No usar kernel trick cuando se lo requiera también.
                -en DT: usar profundidad baja.

               *Overfitting: sino se hace sanitización/regularización de lso datos.
               -en k-nearest-Neighbors: usar un k muy grande.
               -SVM: usar un C muy alto softmargin (permitir pocos errores)
               -en DT: usar profundidad alta y no realizar poda.

- Red neuronal: *Underfitting: muy pocas capas. Poca cant de épocas. Learning rate muy chico.
                *Overfitting: demasiadas capas. Entrenar muchas épocas. falta de regularización. Dataset pequeño (suar data aumentation)
